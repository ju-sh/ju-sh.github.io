Abstract interpretation

  • Foundations laid by the work of Patrick and Radhia Cousot
    • Book: Principles of abstract interpretation
    • https://cs.nyu.edu/~pcousot/
  • A form of static analysis
    • Program is not actually run
  • Examples of absint tools: astrée (used by Airbus)
  • Approximate actual program
  • Give up some precision in exchange for scalability
  • Overapproximation is always safe
    • There are no false negatives for errors, only false positives
    • If absint finds that a design is error free, that will be correct
    • If absint finds that a design has error, that is potentially correct

Galois connection

Abstract and concrete domains are posets.

  • Abstract domain: (A, ≼)
  • Concrete domain: (C, ⊑)
  • α: C → A
    • 'abstractify'
  • γ: A → C
    • 'reify'

The pair (α, γ) constitutes a Galois connection.

∀P ∈ C, ∀P̅ ∈ A,
  α(P) ≼ P̅   ↔  P ⊑ γ(P̅)

This is written as:

        α
(A, ≼)  ⇆  (C, ⊑)
        γ
  • Concrete properties in C are approximated by corresponding properties in A.
