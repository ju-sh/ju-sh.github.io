Mail server stuff

DKIM, DMARC and SPF

  • DKIM, DMARC and SPF work together to make domain impersonation difficult.
    • All three are DNS TXT records ??
  • DKIM  ≥  ≥
    • Domain Keys Identified Mail
    • Helps prevent impersonation of domains.
  • DMARC  ≥:
    • Domain-based Message Authentication Reporting and Conformance
    • Used to prevent email address spoofing
    • Shows that an email really did originate from the server that it appears to originate from.
    • DMARC record is a DNS TXT record that contains a DMARC policy (a non-specialized type of DNS record)
  • SPF  ≥:
    • Sender Policy Framework
    • Helps prevent impersonation of domains.

DNS records

  • DNS TXT record  ≥
    • Allows domain admin to insert arbitrary text data into DNS.
  • DNS MX record  ≥
    • helps redirect mail to mail exchange server
  • DNS CNAME record  ≥
    • Canonical name
    • Points from an alias domain to the actual domain
      • Pointed domain can point to yet another domain though, if that too has a CNAME record.
    • Points to a domain, not an IP address.
  • DNS NS record
    • Name server record
    • If present, it indicates that this server is the authoritative name server for this domain.
  • 1.1.1.1: Cloudflare's public DNS server
  • 8.8.8.8: Google's public DNS server
  • Many kinds of DNS servers (aka 'name server' or 'name resolver' ??)  ≥
    • Recursive name server
    • TLD name server
    • Authoritative name server
  • 'A record' for ipv4 and 'AAAA record' for ipv6
