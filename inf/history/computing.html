FORTRAN.

Was an improvement over assembly language as we didn't need to make up names for intermediate values.

For example[1], we could write x = (a+b) * (c+d) instead of

r1 =  a + b
r2 =  c + d
 x = r1 * r2

Clearly, the one-line version is more readable. The temporary names for intermediate values doesn't add value for understandability.

But the assembly version makes order of evaluation explicit.
